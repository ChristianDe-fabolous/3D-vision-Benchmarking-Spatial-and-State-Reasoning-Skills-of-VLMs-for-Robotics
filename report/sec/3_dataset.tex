\section{Dataset \& Tasks}
\label{sec:dataset}

\subsection{Source Data}
Robo2VLM-1~\cite{chen2025robo2vlm} builds VQA pairs from DROID~\cite{khazatsky2024droid}
trajectories (roughly half of the data) and OXE trajectories (the other half).
\cite{chen2025robo2vlm} use a novel framework to identify keyframes within each
trajectory and pair them with the goal text, ordered action-phase labels, and
sensor-derived success/failure signals captured from three synchronized camera views
(top-left, top-right, wrist). From this pool we select 47 scenes from the DROID subset,
covering diverse manipulation tasks such as placing, screwing, folding, and stacking.

\subsection{Failure-Mode Detection Sub-Tasks}
We define four multiple-choice failure-mode sub-tasks, all evaluated on real DROID/OXE
trajectory frames.

\textbf{Action Phase Identification} (\texttt{action\_phase\_id}): given a single image,
the goal text, and an ordered list of phases, the model selects which phase (A--D) is
currently depicted, or answers ``Cannot be determined'' (E). Random baseline: 33.3\%.

\textbf{Phase Success} (\texttt{phase\_success}): given a single image, the goal text, and
a target phase, the model judges whether that phase is Before, Is currently performing,
After, or Cannot be determined. Random baseline: 25.0\%.

\textbf{Progress Detection} (\texttt{progress}): given two side-by-side images and the
goal text, the model judges whether the task progressed forward, backward, not at all, or
Cannot be determined. Random baseline: 33.3\%.

\textbf{Task Success} (\texttt{task\_success}): given a single image and the goal text,
the model answers Yes (complete), No (failed or incomplete), or Cannot be determined.
Random baseline: 33.3\%.

For \texttt{action\_phase\_id}, \texttt{task\_success}, and \texttt{progress} we
additionally inject an adversarial \emph{Cannot be determined} (CnBD) variant: when
\texttt{special\_image == "random\_scene"}, the input image is replaced with an unrelated
scene, making ``Cannot be determined'' the only correct answer. This variant tests
whether a model detects a domain mismatch between image and goal text, rather than
guessing a plausible-looking phase.

\subsection{Multi-View Consistency}
The same question is additionally posed from three camera views of the same scene:
top-left (TL), top-right (TR), and wrist (W). We measure consistency as whether
independently elicited answers agree across views, using three metrics: TL$\leftrightarrow$TR
agreement, agreement of at least one top view with the wrist view
($\geq$1~Top$\leftrightarrow$W), and All-3-agree, where all three views give the same
answer. Treating each view as an independent binary draw gives baselines of 50\%, 75\%,
and 25\% for the three metrics, respectively.
