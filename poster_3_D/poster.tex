\headerbox{Introduction}{name=intro, column=0, row=0, span=6}{
{\color{myblack}
\textbf{Abstract:} We benchmark Vision-Language Models (VLMs) on two robot-supervision capabilities --- \textbf{multi-view consistency} and \textbf{failure-mode detection} --- using newly designed evaluation questions built on the Robo2VLM-1 [1] dataset. We evaluate 7 models across 3 families (Gemma4, Qwen3, InternVL3).
\\[4pt]

\headerbox{Conclusion}{name=conclusion, column=0, span=3, below=targets}{
\setlength{\itemsep}{2pt}
\setlength{\parskip}{1pt}
{\color{myblack}
\large
\begin{itemize}
  \item \textbf{Overall performance is low.} All 7 models miss the fail at object state and temporal reasoning
  \item \textbf{Scale helps without adaptation.} Gemma4-31B performs best overall (47.5\,\%)
  \item \textbf{Robotic state understanding remains weak.} Off-the-shelf VLMs do not randomly guess some object identity/relative position (pick-and-place) but struggle with state-dependent actions (e.g., \textit{scrunch}, \textit{align}, \textit{close}).
\end{itemize}
}}

\headerbox{Limitations \& Future Work}{name=limitations, column=3, span=3, below=results}{
\setlength{\itemsep}{2pt}
\setlength{\parskip}{1pt}
{\color{myblack}
\large
\begin{itemize}
  \item \textbf{Manual verification limits scale.} Automated pipelines are necessary to extend beyond the current 47 scenes / 6224 questions.
  \item \textbf{Future step.} Use Robo2VLM idea to generate large scale dataset and test task cross difficulty.
\end{itemize}
